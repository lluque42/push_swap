Detailed Description

Base for typedef t_element to represent each number and its sorting metadata.

This type is to be used as the list node content member of each stack.

Field Documentation

◆ cost_a

s_element::cost_a

The number of movements that would be needed to apply to stack A to put it in a state in which, if pushing to A this element, it lands in such a position that maintains stack A in a (circularly) sorted state.

◆ cost_b

s_element::cost_b

The number of movements that would be needed to apply to stack B if this element was to be at the top (so it could be pushed to A).

s_element::total_cost

The total number of ps_lang rotation instructions (affecting stack A, B or both) that would be needed if this element was to be pushed to stack A while maintaining this stack in a (circularly) sorted state.
